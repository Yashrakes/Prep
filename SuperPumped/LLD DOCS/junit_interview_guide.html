🔬 JUnit Fundamentals CORE

⚡ What is JUnit?

JUnit is the standard unit testing framework for Java. JUnit 5 (Jupiter) is the current version. It's modular: junit-jupiter-api for writing tests, junit-jupiter-engine for running them, and junit-vintage-engine for JUnit 4 compatibility.

🔄 JUnit 4 vs JUnit 5

  • JUnit 5 uses @Test from org.junit.jupiter, not org.junit
  • No public modifier needed on test methods in JUnit 5
  • @Before/@After replaced with @BeforeEach/@AfterEach
  • Supports nested tests, parameterized tests, extensions
  • @RunWith replaced with @ExtendWith

📐 The AAA Pattern — Golden Rule of Unit Testing

Every test should follow Arrange → Act → Assert. Arrange sets up data/mocks. Act calls the method under test. Assert verifies the result. This is the most important pattern interviewers look for.

Java — AAA Pattern
import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

class CalculatorTest {

    @Test
    void add_ShouldReturnSumOfTwoNumbers() {
        // ARRANGE — set up inputs
        Calculator calculator = new Calculator();
        int a = 5, b = 3, expected = 8;

        // ACT — call the method under test
        int result = calculator.add(a, b);

        // ASSERT — verify the outcome
        assertEquals(expected, result, "5 + 3 should equal 8");
    }

    @Test
    void divide_ShouldThrowException_WhenDivisorIsZero() {
        Calculator calculator = new Calculator();

        // assertThrows captures the exception for further assertion
        ArithmeticException ex = assertThrows(
            ArithmeticException.class,
            () -> calculator.divide(10, 0)
        );
        assertEquals("/ by zero", ex.getMessage());
    }
}

📊 Types of Testing in Spring Boot

  • Unit Tests: Test one class in isolation. Use Mockito to mock dependencies. No Spring context, milliseconds fast.
  • Slice Tests: @WebMvcTest, @DataJpaTest — load only a relevant layer of the context.
  • Integration Tests: Load full context with @SpringBootTest. Test multiple layers together.
  • E2E Tests: Real HTTP with TestRestTemplate or Testcontainers for real DB.

🏷️ Annotations Deep Dive ALL ANNOTATIONS

@Test

Marks a method as a test. In JUnit 5, the method does not need to be public — package-private works. The test fails if any exception is thrown, passes if it completes normally. Putting it on a private method causes a JUnit error.

Java — Lifecycle Annotations
@TestInstance(TestInstance.Lifecycle.PER_CLASS) // one instance for all tests
class UserServiceTest {

    private UserService userService;

    /**
     * @BeforeAll — runs ONCE before any test in the class.
     * Must be static by default. With PER_CLASS it can be an instance method.
     * Use for expensive one-time setup: DB connections, schema creation.
     */
    @BeforeAll
    void initAll() {
        System.out.println("Runs once before all tests");
    }

    /**
     * @BeforeEach — runs before EVERY test method.
     * Key use: re-create objects to prevent test pollution (shared state bugs).
     */
    @BeforeEach
    void setUp() {
        userService = new UserService(); // fresh instance each time
    }

    @Test
    @DisplayName("Should return user when valid ID given")
    void findUser() { /* ... */ }

    @Test
    @Disabled("Bug #1234 — fix pending, skipping to unblock CI")
    void skippedTest() { /* ... */ }

    /**
     * @AfterEach — runs after every test.
     * Use for cleanup: closing files, resetting state, clearing DB rows.
     */
    @AfterEach
    void tearDown() { System.out.println("After each test"); }

    /**
     * @AfterAll — runs ONCE after all tests are done.
     * Use for global cleanup: stopping servers, closing pool connections.
     */
    @AfterAll
    void tearDownAll() { System.out.println("Final cleanup"); }
}
Java — Parameterized Tests (Very Common Interview Topic)
import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.*;

class ParameterizedExamples {

    // @ValueSource — single-value injection for simple cases
    @ParameterizedTest
    @ValueSource(strings = {"yash@example.com", "admin@bank.com"})
    void validEmail_ShouldPass(String email) {
        assertTrue(EmailValidator.isValid(email));
    }

    // @CsvSource — multiple columns per row (great for input/output pairs)
    @ParameterizedTest
    @CsvSource({
        "5, 3, 8",
        "10, 0, 10",
        "-1, 1, 0"
    })
    void add_ShouldReturnCorrectSum(int a, int b, int expected) {
        assertEquals(expected, new Calculator().add(a, b));
    }

    // @MethodSource — for complex objects that can't be expressed in CSV
    @ParameterizedTest
    @MethodSource("provideUsers")
    void user_ShouldBeValid(User user, boolean expectedValid) {
        assertEquals(expectedValid, UserValidator.isValid(user));
    }

    static Stream<Arguments> provideUsers() {
        return Stream.of(
            Arguments.of(new User("Yash", "yash@test.com"), true),
            Arguments.of(new User("", ""), false)
        );
    }

    // @NullAndEmptySource — tests null AND empty string as inputs automatically
    @ParameterizedTest
    @NullAndEmptySource
    void nullOrEmpty_ShouldFailValidation(String value) {
        assertFalse(StringValidator.isValid(value));
    }

    // @EnumSource — test with all or selected enum values
    @ParameterizedTest
    @EnumSource(value = UserRole.class, names = {"ADMIN", "USER"})
    void activeRoles_ShouldHavePermissions(UserRole role) {
        assertNotNull(PermissionService.getPermissions(role));
    }
}

class OrderServiceTest {

    // flaky/timing-sensitive logic or verifying idempotency.
    @RepeatedTest(5)
    void idempotentOperation_ShouldAlwaysReturnSame() { /* ... */ }

    // @Tag — categorize tests so CI can run specific groups.
    // In maven: mvn test -Dgroups="smoke" to run only smoke tests.
    @Test
    @Tag("smoke")
    void smokeTest() { /* ... */ }

    // @EnabledOnOs, @EnabledIfEnvironmentVariable — conditional execution
    @Test
    @EnabledIfEnvironmentVariable(named = "ENV", matches = "PROD")
    void prodOnlyTest() { /* ... */ }

    // Without it, the first failure stops the rest from running.
    @Test
    void groupedAssertions() {
        User user = userService.getUser(1L);
        assertAll(
            "user properties",
            () -> assertNotNull(user),
            () -> assertEquals("Yash", user.getName()),
            () -> assertEquals("yash@test.com", user.getEmail()),
            () -> assertNotNull(user.getCreatedAt())
        );
    }

🎭 Mockito & Mocking ESSENTIAL

🤔 Why Mock?

Unit tests must test ONE class in isolation. If your OrderService depends on UserRepository and EmailService, you don't want to hit the real DB or send real emails in a unit test. Mocks replace those dependencies with controlled fakes so you test only the logic in OrderService. This is what makes unit tests fast and deterministic.

Mock vs Stub vs Spy

  • Mock: Fully fake object. All methods return defaults (null/0/false). You verify interactions on it.
  • Stub: A mock with pre-configured return values via when(...).thenReturn(...).
  • Spy: Wraps a REAL object. Calls real methods unless you override specific ones.

@Mock vs @InjectMocks

  • @Mock — Creates a mock of the annotated type
  • @InjectMocks — Creates the real object AND injects all @Mocks into it
  • Always pair with @ExtendWith(MockitoExtension.class)
  • Injection order: Constructor → Setter → Field
Java — Complete Mockito Patterns
import org.junit.jupiter.api.extension.ExtendWith;
import org.mockito.*;
import org.mockito.junit.jupiter.MockitoExtension;
import static org.mockito.Mockito.*;
import static org.mockito.ArgumentMatchers.*;

/**
 * @ExtendWith(MockitoExtension.class):
 *  - Activates @Mock, @InjectMocks, @Spy, @Captor annotations
 *  - Auto-closes mocks after each test (no memory leaks)
 *  - Enables STRICT_STUBS by default (fails on unused stubs)
 *  - JUnit 5 replacement for JUnit 4's @RunWith(MockitoJUnitRunner.class)
 */
@ExtendWith(MockitoExtension.class)
class OrderServiceTest {

    @Mock private UserRepository userRepository;
    @Mock private EmailService emailService;

    // Creates a REAL OrderService and injects the mocks above into it
    @InjectMocks private OrderService orderService;

    // Captures the exact argument passed to a mocked method
    @Captor private ArgumentCaptor<Order> orderCaptor;

    @Test
    void placeOrder_HappyPath() {
        // STUB — define what the mock returns when called
        User user = new User(1L, "Yash", "yash@test.com");
        when(userRepository.findById(anyLong()))
            .thenReturn(Optional.of(user));

        // doNothing() — for void methods (can't use thenReturn for void)
        doNothing().when(emailService).send(anyString(), anyString());

        Order placed = orderService.placeOrder(1L, "Laptop");

        assertNotNull(placed);
        assertEquals("Laptop", placed.getProductName());

        // VERIFY — confirm the mock was called with expected arguments
        verify(userRepository, times(1)).findById(1L);
        verify(emailService).send(eq("yash@test.com"), contains("Confirmed"));

        // Ensure no unexpected interactions happened
        verifyNoMoreInteractions(userRepository, emailService);
    }

    @Test
    void placeOrder_ShouldCaptureObjectPassedToSave() {
        when(userRepository.findById(anyLong()))
            .thenReturn(Optional.of(new User(1L, "Yash", "y@t.com")));

        orderService.placeOrder(1L, "Phone");

        // Capture the Order object passed to save() and inspect its fields
        verify(userRepository).save(orderCaptor.capture());
        Order captured = orderCaptor.getValue();
        assertEquals("Phone", captured.getProductName());
        assertEquals(OrderStatus.PENDING, captured.getStatus());
    }

    @Test
    void placeOrder_ShouldThrow_WhenUserNotFound() {
        // thenThrow — simulate exceptions from dependencies
        when(userRepository.findById(anyLong()))
            .thenThrow(new UserNotFoundException("Not found"));

        assertThrows(UserNotFoundException.class,
            () -> orderService.placeOrder(99L, "Laptop"));

        // Email must NEVER be called when user lookup fails
        verify(emailService, never()).send(any(), any());
    }

    @Test
    void spy_ShouldCallRealMethod_UnlessStubbed() {
        List<String> realList = new ArrayList<>();
        List<String> spyList = spy(realList);

        spyList.add("Yash"); // real add() is called

        // doReturn() syntax required for spies to avoid calling real method during setup
        doReturn(100).when(spyList).size();

        assertEquals(1, realList.size()); // real list unchanged
        assertEquals(100, spyList.size()); // stubbed on spy
    }

    // thenAnswer — return different values based on input arguments dynamically
    @Test
    void thenAnswer_DynamicReturnValue() {
        when(userRepository.findById(anyLong())).thenAnswer(invocation -> {
            Long id = invocation.getArgument(0);
            return id == 1L
                ? Optional.of(new User(1L, "Yash"))
                : Optional.empty();
        });
        assertTrue(userRepository.findById(1L).isPresent());
        assertFalse(userRepository.findById(99L).isPresent());
    }
}
💡 Key Distinction: Use when(mock.method()).thenReturn() for normal stubs. Use doReturn().when(mock).method() for void methods and spies. The latter form does NOT call the real method during setup — critical for spies.

⚙️ Service Layer Tests SPRING BOOT

🎯 Strategy for Service Tests

Service tests should be pure unit tests — no Spring context, no real DB. Use @ExtendWith(MockitoExtension.class) and mock all repositories and downstream services. This keeps them blazing fast (milliseconds) and makes failures crystal-clear because there's only one real class in play.

Java — Service Under Test
@Service
public class UserService {

    private final UserRepository userRepository;
    private final PasswordEncoder passwordEncoder;
    private final EmailService emailService;

    // Constructor injection — BEST practice. Makes dependencies explicit
    // and allows testing without Spring or reflection.
    public UserService(UserRepository userRepository,
                       PasswordEncoder passwordEncoder,
                       EmailService emailService) {
        this.userRepository = userRepository;
        this.passwordEncoder = passwordEncoder;
        this.emailService = emailService;
    }

    public UserDTO createUser(CreateUserRequest request) {
        if (userRepository.existsByEmail(request.getEmail())) {
            throw new DuplicateEmailException("Email already registered");
        }
        User user = User.builder()
            .name(request.getName())
            .email(request.getEmail())
            .password(passwordEncoder.encode(request.getPassword()))
            .role(Role.USER)
            .build();
        User saved = userRepository.save(user);
        emailService.sendWelcome(saved.getEmail());
        return UserMapper.toDTO(saved);
    }

    public UserDTO getUserById(Long id) {
        return userRepository.findById(id)
            .map(UserMapper::toDTO)
            .orElseThrow(() -> new UserNotFoundException("User not found: " + id));
    }
}
Java — Complete Service Unit Test (All Scenarios)
@ExtendWith(MockitoExtension.class)
class UserServiceTest {

    @Mock private UserRepository userRepository;
    @Mock private PasswordEncoder passwordEncoder;
    @Mock private EmailService emailService;
    @InjectMocks private UserService userService;
    @Captor private ArgumentCaptor<User> userCaptor;

    // ===== HAPPY PATH =====

    @Test
    @DisplayName("createUser — should hash password, save user, and send welcome email")
    void createUser_HappyPath() {
        // Arrange
        CreateUserRequest req = new CreateUserRequest("Yash", "yash@test.com", "pass123");
        when(userRepository.existsByEmail("yash@test.com")).thenReturn(false);
        when(passwordEncoder.encode("pass123")).thenReturn("$2a$hashed");
        User savedUser = User.builder().id(1L).name("Yash")
                          .email("yash@test.com").role(Role.USER).build();
        when(userRepository.save(any(User.class))).thenReturn(savedUser);

        // Act
        UserDTO result = userService.createUser(req);

        // Assert — use assertAll so ALL assertions are checked even if one fails
        assertAll(
            () -> assertNotNull(result),
            () -> assertEquals("Yash", result.getName()),
            () -> assertEquals("yash@test.com", result.getEmail())
        );

        // Capture the User passed to save() — verify its internal state
        verify(userRepository).save(userCaptor.capture());
        User captured = userCaptor.getValue();
        assertEquals("$2a$hashed", captured.getPassword()); // was password hashed?
        assertEquals(Role.USER, captured.getRole());         // correct role?

        verify(emailService, times(1)).sendWelcome("yash@test.com");
    }

    // ===== EDGE CASE — Duplicate email =====

    @Test
    @DisplayName("createUser — should throw DuplicateEmailException if email exists")
    void createUser_DuplicateEmail_ThrowsException() {
        CreateUserRequest req = new CreateUserRequest("Yash", "dup@test.com", "pass");
        when(userRepository.existsByEmail("dup@test.com")).thenReturn(true);

        DuplicateEmailException ex = assertThrows(
            DuplicateEmailException.class,
            () -> userService.createUser(req)
        );
        assertTrue(ex.getMessage().contains("already registered"));

        // CRITICAL: verify side effects did NOT happen on failure
        verify(userRepository, never()).save(any());
        verify(emailService, never()).sendWelcome(any());
    }

    // ===== USER NOT FOUND =====

    @Test
    void getUserById_ShouldThrow_WhenNotFound() {
        when(userRepository.findById(99L)).thenReturn(Optional.empty());

        assertThrows(UserNotFoundException.class,
            () -> userService.getUserById(99L));
    }

    @Test
    void getUserById_ShouldReturnDTO_WhenFound() {
        User user = new User(1L, "Yash", "yash@test.com");
        when(userRepository.findById(1L)).thenReturn(Optional.of(user));

        UserDTO dto = userService.getUserById(1L);

        assertEquals("Yash", dto.getName());
        assertEquals("yash@test.com", dto.getEmail());
    }
}
🌐 Controller / REST API Tests SPRING BOOT

@WebMvcTest — The Right Tool for Controller Tests

@WebMvcTest is a slice test that loads only the web layer: controllers, filters, @ControllerAdvice, WebMvcConfigurer. It does NOT load service or repository beans. You must provide the service as a @MockBean. It uses an in-memory mock servlet — no real HTTP server starts, making it fast and focused.

@MockBean vs @Mock

@Mock is a pure Mockito annotation — no Spring involvement. @MockBean creates a Mockito mock AND registers it as a Spring bean, replacing any existing bean of that type in the context. In @WebMvcTest you MUST use @MockBean for the service, because the controller is wired by Spring and needs a Spring-managed bean.

MockMvc vs TestRestTemplate

MockMvc tests without a real HTTP server. It's fast and used with @WebMvcTest. TestRestTemplate sends real HTTP over a socket. Used with @SpringBootTest(webEnvironment = RANDOM_PORT) for full-stack integration tests. For controller unit tests, always prefer MockMvc.

Java — Controller Under Test
@RestController
@RequestMapping("/api/v1/users")
public class UserController {

    private final UserService userService;

    public UserController(UserService userService) {
        this.userService = userService;
    }

    @PostMapping
    public ResponseEntity<UserDTO> createUser(@RequestBody @Valid CreateUserRequest req) {
        return ResponseEntity.status(HttpStatus.CREATED).body(userService.createUser(req));
    }

    @GetMapping("/{id}")
    public ResponseEntity<UserDTO> getUserById(@PathVariable Long id) {
        return ResponseEntity.ok(userService.getUserById(id));
    }

    @DeleteMapping("/{id}")
    public ResponseEntity<Void> deleteUser(@PathVariable Long id) {
        userService.deleteUser(id);
        return ResponseEntity.noContent().build();
    }
}
Java — @WebMvcTest Controller Test (Full)
import org.springframework.boot.test.autoconfigure.web.servlet.WebMvcTest;
import org.springframework.boot.test.mock.mockito.MockBean;
import org.springframework.test.web.servlet.MockMvc;
import org.springframework.http.MediaType;
import static org.springframework.test.web.servlet.request.MockMvcRequestBuilders.*;
import static org.springframework.test.web.servlet.result.MockMvcResultMatchers.*;
import static org.mockito.Mockito.*;

/**
 * @WebMvcTest(UserController.class) — loads ONLY the web layer.
 * No service beans, no DB beans. Extremely fast slice test.
 */
@WebMvcTest(UserController.class)
class UserControllerTest {

    // MockMvc is auto-configured and injected by @WebMvcTest
    @Autowired private MockMvc mockMvc;

    // @MockBean — creates Mockito mock AND registers it in Spring context
    @MockBean private UserService userService;

    // ObjectMapper for serializing request bodies to JSON
    @Autowired private ObjectMapper objectMapper;

    // ===== POST — Create User =====

    @Test
    @DisplayName("POST /users — should return 201 with created user")
    void createUser_ShouldReturn201() throws Exception {
        CreateUserRequest req = new CreateUserRequest("Yash", "yash@test.com", "pass123");
        UserDTO mockResponse = new UserDTO(1L, "Yash", "yash@test.com");

        when(userService.createUser(any())).thenReturn(mockResponse);

        mockMvc.perform(
            post("/api/v1/users")
                .contentType(MediaType.APPLICATION_JSON)
                .content(objectMapper.writeValueAsString(req))
        )
        .andExpect(status().isCreated())                       // 201
        .andExpect(content().contentType(MediaType.APPLICATION_JSON))
        .andExpect(jsonPath("$.id").value(1))
        .andExpect(jsonPath("$.name").value("Yash"))
        .andExpect(jsonPath("$.email").value("yash@test.com"));

        verify(userService, times(1)).createUser(any());
    }

    // ===== GET — Found =====

    @Test
    void getUserById_ShouldReturn200_WhenFound() throws Exception {
        when(userService.getUserById(1L))
            .thenReturn(new UserDTO(1L, "Yash", "yash@test.com"));

        mockMvc.perform(get("/api/v1/users/1"))
            .andExpect(status().isOk())
            .andExpect(jsonPath("$.name").value("Yash"));
    }

    // ===== GET — Not Found (tests @ControllerAdvice handling) =====

    @Test
    void getUserById_ShouldReturn404_WhenNotFound() throws Exception {
        when(userService.getUserById(99L))
            .thenThrow(new UserNotFoundException("User not found"));

        mockMvc.perform(get("/api/v1/users/99"))
            .andExpect(status().isNotFound()); // @ControllerAdvice maps this
    }

    // ===== Validation — Bad Request =====

    @Test
    void createUser_ShouldReturn400_WhenBodyInvalid() throws Exception {
        // Empty JSON body — @Valid on @RequestBody will fail
        mockMvc.perform(
            post("/api/v1/users")
                .contentType(MediaType.APPLICATION_JSON)
                .content("{}")
        )
        .andExpect(status().isBadRequest());

        // Service should NEVER be called if Bean Validation fails
        verify(userService, never()).createUser(any());
    }

    // ===== DELETE =====

    @Test
    void deleteUser_ShouldReturn204() throws Exception {
        doNothing().when(userService).deleteUser(1L);

        mockMvc.perform(delete("/api/v1/users/1"))
            .andExpect(status().isNoContent()); // 204 No Content
    }

    // ===== Testing with Request Headers / Params =====

    @Test
    void getUsersWithQueryParam() throws Exception {
        mockMvc.perform(
            get("/api/v1/users")
                .param("page", "0")
                .param("size", "10")
                .header("Authorization", "Bearer test-token")
        )
        .andExpect(status().isOk());
    }
}

class CoreJavaTests {

    // ===== Collections =====
    @Test
    void testListContents() {
        List<String> names = List.of("Yash", "Rahul", "Priya");
        assertEquals(3, names.size());
        assertTrue(names.contains("Yash"));
        // assertIterableEquals checks contents AND order
        assertIterableEquals(List.of("Yash", "Rahul", "Priya"), names);
    }

    // ===== Streams =====
    @Test
    void testStreamFiltering() {
        List<Integer> numbers = List.of(1, 2, 3, 4, 5, 6);
        List<Integer> evens = numbers.stream()
            .filter(n -> n % 2 == 0)
            .collect(Collectors.toList());
        assertEquals(List.of(2, 4, 6), evens);
    }

    // ===== Optional =====
    @Test
    void testOptional_Empty_ThrowsOnGet() {
        Optional<String> empty = Optional.empty();
        assertFalse(empty.isPresent());
        // Verify orElse, orElseThrow work correctly
        assertEquals("default", empty.orElse("default"));
        assertThrows(NoSuchElementException.class, empty::get);
    }

    // ===== Custom equals/hashCode — critical for asserting domain objects =====
    @Test
    void testEquality() {
        User u1 = new User(1L, "Yash");
        User u2 = new User(1L, "Yash");
        // assertEquals uses .equals() — you MUST override equals() in User
        assertEquals(u1, u2);
        // Contract: equal objects must have equal hashCodes
        assertEquals(u1.hashCode(), u2.hashCode());
    }

    // ===== Exception chaining =====
    @Test
    void testExceptionCause() {
        RuntimeException ex = assertThrows(RuntimeException.class, () -> {
            try {
                Integer.parseInt("not-a-number");
            } catch (NumberFormatException e) {
                throw new RuntimeException("Parse failed", e);
            }
        });
        // Verify the wrapped cause type
        assertInstanceOf(NumberFormatException.class, ex.getCause());
    }

    // ===== Thread Safety Test — AtomicInteger =====
    @Test
    void testAtomicCounter_ThreadSafety() throws InterruptedException {
        AtomicInteger counter = new AtomicInteger(0);
        int threadCount = 100;
        CountDownLatch latch = new CountDownLatch(threadCount);

        for (int i = 0; i < threadCount; i++) {
            new Thread(() -> {
                counter.incrementAndGet(); // atomic — no race condition
                latch.countDown();
            }).start();
        }

        latch.await(); // wait for all threads to complete
        assertEquals(threadCount, counter.get());
    }

    @Test
    void testConcurrentHashMap_NoLostUpdates() throws Exception {
        ConcurrentHashMap<String, AtomicInteger> map = new ConcurrentHashMap<>();
        map.put("key", new AtomicInteger(0));

        ExecutorService executor = Executors.newFixedThreadPool(10);
        CountDownLatch latch = new CountDownLatch(100);

        for (int i = 0; i < 100; i++) {
            executor.submit(() -> {
                map.get("key").incrementAndGet();
                latch.countDown();
            });
        }
        latch.await();
        executor.shutdown();
        assertEquals(100, map.get("key").get());
    }

🎯 Interview Q&A ALL LEVELS
EASY What is the difference between @Mock and @Spy in Mockito? ▶

@Mock creates a completely fake object. None of the real class's code ever runs. All methods return default values (null, 0, false) unless stubbed. You use it to isolate the class under test from its dependencies.

@Spy wraps a real object. By default, real methods execute when you call them on a spy. You selectively override specific methods using doReturn().when(spy).method(). Critical: always use doReturn() syntax with spies, not when().thenReturn(), because the latter actually invokes the real method during setup which can cause unintended side effects.

EASY What is @InjectMocks and how does Mockito decide how to inject? ▶

Mockito creates an instance of the annotated class and injects all available @Mock and @Spy fields into it. The injection order is: (1) Constructor injection — tries the largest constructor whose parameter types match available mocks; (2) Setter injection — looks for setters matching mock types; (3) Field injection — directly sets private fields via reflection.

Constructor injection is preferred in production code because it makes dependencies explicit and enables testing without any framework magic at all — you can just call new UserService(mockRepo, mockEmail).

MEDIUM What is the difference between @WebMvcTest and @SpringBootTest? ▶

@WebMvcTest is a slice test that loads only the web layer — controllers, filters, @ControllerAdvice, WebMvcConfigurer. It does NOT load service or repository beans. It uses MockMvc with an in-memory servlet engine (no real HTTP server). It's fast and purpose-built for testing controller logic, request mapping, and HTTP response codes.

@SpringBootTest loads the entire application context — every bean, all auto-configurations. With webEnvironment = RANDOM_PORT, it starts a real embedded HTTP server and you test via TestRestTemplate. It's slow but validates the full stack. Use it for integration tests, not controller unit tests.

MEDIUM Why should you prefer constructor injection from a testability standpoint? ▶

With field injection (@Autowired private UserRepo repo), you cannot create the class in a unit test without Spring or reflection because the field is set by Spring. You're forced to use @ExtendWith(SpringExtension.class) or resort to ReflectionTestUtils.setField(), both of which are heavier.

With constructor injection, you simply call new UserService(mockRepo, mockEmail). No Spring, no reflection, pure Java. Your test is a plain JUnit + Mockito test that runs in milliseconds. Constructor injection also makes it impossible to instantiate the class without providing all dependencies, which catches wiring errors at compile time rather than at runtime.

MEDIUM What is ArgumentCaptor and when would you use it? ▶

ArgumentCaptor captures the exact argument that was passed to a mocked method during the test execution so you can assert on that argument afterward. You use it when the object was created internally inside the method under test and you can't get a reference to it from the outside.

Classic example: verifying that userRepository.save() was called with a User whose password was hashed and whose role was set to USER. Without ArgumentCaptor, you'd have to use any(User.class) which tells you nothing about the object's internal state. With it, you can call captor.getValue() after verify() and run full assertions on the captured object.

HARD What is strict stubbing in Mockito and why is it the default? ▶

Strict stubbing (Strictness.STRICT_STUBS, default in MockitoExtension) enforces two rules. First, every stub you define must be actually used by the test — if you define when(repo.findById(1L)).thenReturn(...) but the code never calls findById, the test fails with "unnecessary stubbing". Second, argument matching is stricter — a stub for findById(1L) will NOT satisfy a call to findById(2L).

This catches two important bugs: tests that have stubs for code paths that are never actually exercised (suggesting the test or the stub is wrong), and argument mismatches where the code calls the method with different arguments than you expected (suggesting a logic bug). It forces tests to be precise about what they're testing.

HARD How do you test a method that calls LocalDateTime.now() or a static utility? ▶

The cleanest, most testable approach is to inject a Clock. Instead of calling LocalDateTime.now() directly, inject java.time.Clock as a dependency and call LocalDateTime.now(clock). In production, Spring provides @Bean Clock clock() { return Clock.systemDefaultZone(); }. In tests, you inject Clock.fixed(someInstant, UTC), which gives you full control over "now" without any mocking framework.

The alternative is MockedStatic from Mockito 3.4+: wrap your test in a try-with-resources block using mockStatic(LocalDateTime.class), stub the static method inside, and the static mock is automatically restored when the block exits. PowerMock was the old approach but is now legacy and adds significant complexity.

HARD What is @TestInstance(PER_CLASS) and when would you use it? ▶

By default, JUnit 5 creates a new instance of the test class for every single test method (PER_METHOD). This prevents test pollution — instance variables can't leak state between tests. The side effect is that @BeforeAll and @AfterAll methods must be static.

With @TestInstance(PER_CLASS), one shared instance is used for all test methods in the class. This allows @BeforeAll/@AfterAll to be instance methods (useful if they reference non-static fields). Common use cases: Testcontainers integration tests where you want to start one DB container for all tests in the class rather than one per test; @Nested test classes that need to share outer class state.

HARD How do you test Spring Security-secured endpoints in @WebMvcTest? ▶

By default, @WebMvcTest loads Spring Security. Requests without credentials return 401 or 403. You have several options. The simplest is @WithMockUser(username="yash", roles="USER") on the test method — this injects a mock authenticated principal into the security context without needing a real JWT or session.

For JWT-based auth, you can either: (1) mock your token validation filter/bean so it always passes, (2) use @WithMockUser alongside a mocked JwtService bean, or (3) if you're not testing security at all, exclude it with @WebMvcTest(excludeAutoConfiguration = SecurityAutoConfiguration.class). Security-specific behavior (role-based access, method security) is typically tested in a separate dedicated security test class.

MEDIUM What test coverage percentage should you target and why? ▶

Coverage percentage is a proxy metric — high coverage with zero meaningful assertions is worthless, while 70% coverage of the right code can be excellent. Most mature teams target 80% line or branch coverage for the service layer, with lower targets for infrastructure code.

What actually matters: test every business rule, every exception path, every boundary condition (null, empty, max value). Skip generated code (Lombok, MapStruct), simple DTOs, main() methods, and configuration classes. Use Jacoco with exclude rules in your Maven/Gradle config to filter these out of reporting. The interview answer that stands out: "I focus coverage on the logic that could fail in production and hurt users, not on hitting a number."
